# AI Engineer — Curriculum App

Personal learning app. Left sidebar is an 8-category curriculum tree (foundations → LLMOps). Pick a subtopic, Opus 4.7 generates the full learning package (overview + 3 concepts + 1 hands-on lab). Work through concepts, attempt Q&A per concept (Sonnet 4.6), then run the lab (with 2 free-compute options and a lab-scoped ask-bar). On subtopic completion, generate a Substack post + LinkedIn/X teaser. Progress persists in localStorage; expensive generated content is backed by a gitignored local file cache.

## Files

| File | Purpose |
|---|---|
| `index.html` | The app — markup, styles, all JS in one file |
| `proxy.py` | Local Python proxy — Anthropic API + file cache endpoints |
| `start.bat` / `stop.bat` | Starts/stops proxy + page server |
| `apikey.txt` | Anthropic API key — gitignored |
| `config.js` (from template) | Local config — gitignored |
| `cache/` | Generated subtopic content + write-ups — gitignored, survives browser clears |
| `index_daily.bak.html` | Archived old daily-brief app (predecessor) |

## Architecture

```
start.bat
  ├── proxy.py → localhost:5001  (ANTHROPIC_KEY, Claude API + /cache read/write)
  ├── python -m http.server 8080 (serves index.html)
  └── opens http://localhost:8080/index.html
```

Proxy is pure Python stdlib. No pip installs. CORS-enabled for browser calls.

## Data flow

```
User clicks subtopic in sidebar
  └── Check localStorage (instant)
      → hit: render immediately
      → miss: check cache/{subId}.json via proxy (survives browser clears)
              → hit: hydrate localStorage, render
              → miss: Opus 4.7 → JSON {overview, 3 concepts, 1 lab}
                      → save to localStorage + cache/{subId}.json

User attempts Q&A on a concept
  └── generateQA() → Sonnet 4.6 → 3 questions (cached in localStorage)
  └── user answers → evalQA() → Sonnet 4.6 → {score, feedback, passed}

User chats in lab ask-bar
  └── askLab() → Sonnet 4.6 + cache_control on lab system prompt
                 (Anthropic prompt caching — cheap follow-ups within 5 min)

User completes subtopic (all concepts passed + lab done)
  └── "Generate post" → Opus 4.7 → Substack post (700-1000 words, markdown)
                                  + LinkedIn/X teaser (80-120 words)
      → saved to localStorage + cache/{subId}.json alongside content
```

## Proxy endpoints

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/` (any) | Forward to `api.anthropic.com/v1/messages` |
| `GET` | `/cache?id=` | Read `cache/{id}.json` |
| `POST` | `/cache` | Write `cache/{id}.json` — body: `{id, data}` |
| `GET` | `/rss?url=` | RSS/Atom feed proxy (CORS-safe) |

## Claude calls

| When | Function | Model | Cache strategy |
|---|---|---|---|
| Subtopic first open | `generateSubtopic` | claude-opus-4-7 | localStorage + `cache/{id}.json` (permanent) |
| Subtopic completed | `generateWriteup` | claude-opus-4-7 | localStorage + `cache/{id}.json` (appended to same file) |
| Concept Q&A first | `generateQA` | claude-sonnet-4-6 | localStorage per concept |
| Q&A submit | `evalQA` | claude-sonnet-4-6 | No cache (user-specific) |
| Lab ask-bar turn | `askLab` | claude-sonnet-4-6 | Anthropic prompt caching on lab content (5-min TTL) |

## Lab structure

- Scoped to subtopic; covers all 3 concepts.
- **2 options** — same goal, different tooling (e.g., Colab + LangChain + Chroma vs Kaggle + LlamaIndex + Qdrant).
- **Prerequisites checklist** — must acknowledge before Start button activates.
- **Steps** numbered, each with `do` / `code` / `verify`.
- **Verification** block — concrete checks so learner knows when they're done.
- **Ask-bar** scoped to lab — system prompt hard-binds to lab; off-topic questions redirected.

## Write-up (on subtopic completion)

Triggered by "Generate post" button on the completed subtopic page.

- **Substack post**: 700–1000 words, markdown H2 sections, first-person, opens with the surprising takeaway, closes with "what I'll do differently"
- **LinkedIn/X teaser**: 80–120 words, stands alone, ends with `[link]` placeholder
- Voice: Hamel Husain directness + Simon Willison pragmatism + Eugene Yan clarity
- Readability constraints baked into prompt: plain English → specific → so-what; one-read test; no dense compressions
- Copy buttons for each. Regenerate resets and re-calls Opus.

## File cache (`cache/` folder)

Each `cache/{subId}.json` stores the expensive Opus output only:
```json
{
  "content": { "overview": {}, "concepts": [], "lab": {} },
  "writeup":  { "substack": {}, "social": {} }
}
```

User progress (Q&A answers, lab status, chat history) stays in localStorage only — it's personal and small. Cache files are gitignored; copy the folder to carry generated content to another machine.

## Refresh / force-regenerate

The "Regenerate" button on any subtopic or write-up card sets `forceRegen=true`, which skips both localStorage and file cache and calls Opus fresh. Costs ~$0.80 per subtopic.

## To extend

**Add a subtopic:** add an entry to `CURRICULUM` — the Opus generator handles the rest on first click.

**Change voice/quality:** edit the system prompt inside `generateSubtopic()`.

**Swap models:** edit `OPUS` / `SONNET` constants at the top of the script.

**Update source list:** edit `SOURCES` const. Must match `memory/reference_learning_catalog.md`.

**Add a proxy endpoint:** add routing to `do_GET` or `do_POST` in `proxy.py`.
