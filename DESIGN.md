# AI Engineer — Curriculum App

Personal learning app. Left sidebar is an 8-category curriculum tree (foundations → LLMOps). Pick a subtopic, Opus 4.7 generates the full learning package (overview + 3 concepts + 1 hands-on lab). Work through concepts, attempt Q&A per concept (Sonnet 4.6), then run the lab (with 2 free-compute options and a lab-scoped ask-bar). Completion marks persist in localStorage.

## Files

| File | Purpose |
|---|---|
| `index.html` | The app — markup, styles, all JS in one file |
| `proxy.py` | Local Python proxy — holds Anthropic API key, forwards `/v1/messages` |
| `start.bat` / `stop.bat` | Starts/stops proxy + page server |
| `apikey.txt` | Anthropic API key — gitignored |
| `config.js` (from template) | Local config — gitignored |
| `index_daily.bak.html` | Archived old daily-brief app (predecessor) |

## Architecture

```
start.bat
  ├── proxy.py → localhost:5001  (holds ANTHROPIC_KEY, forwards any POST to api.anthropic.com/v1/messages)
  ├── python -m http.server 8080 (serves index.html)
  └── opens http://localhost:8080/index.html
```

Proxy is pure Python stdlib. No pip installs. CORS-enabled for browser calls.

## Data flow

```
User clicks subtopic in sidebar
  └── S.subtopics[subId].content empty?
        Yes → generateSubtopic() → Opus 4.7 → JSON { overview, 3 concepts, 1 lab }
              cache in localStorage (permanent)
        No  → use cached content
  └── render subtopic view: overview → concept tabs → lab

User attempts Q&A on a concept
  └── generateQA() → Sonnet 4.6 → 3 questions (cached per concept)
  └── user answers → evalQA() → Sonnet 4.6 → {score, feedback, passed}

User chats in lab ask-bar
  └── askLab() → Sonnet 4.6 with lab content in system prompt + cache_control
                 (Anthropic prompt caching — 5-min TTL, dramatic cost drop on follow-ups)
```

## Claude calls

| When | Function | Model | Why this model | Cache? |
|---|---|---|---|---|
| Subtopic first open | `generateSubtopic` | claude-opus-4-7 | Highest quality — learning content must be senior-grade | localStorage, permanent |
| Concept Q&A first | `generateQA` | claude-sonnet-4-6 | 3 short questions — cheaper model fine | localStorage per concept |
| Q&A submit | `evalQA` | claude-sonnet-4-6 | Scoring logic, not generation | No (user-specific) |
| Lab ask-bar turn | `askLab` | claude-sonnet-4-6 | Conversational, lab-scoped | Anthropic prompt caching on lab content |

## Lab structure

- Scoped to subtopic, covers multiple concepts.
- **2 options** shown as tabs — same learning goal, different tooling (e.g., Option A: Colab + LangChain + Chroma vs Option B: Kaggle + LlamaIndex + Qdrant).
- **Prerequisites block** above options — must acknowledge before Start.
- **Steps** numbered, each with `do` / `code` / `verify`.
- **Verification** list at bottom — concrete checks so learner knows they're done.
- **Ask-bar** scoped to lab — system prompt hard-binds to lab content; off-topic questions redirected.

## State (localStorage key: `aieng_v1`)

```js
{
  v: 1,
  subtopics: {
    "rag.chunking": {
      content: { overview, concepts[], lab },      // Opus output, permanent cache
      concepts: {
        c1: {
          questions: [{id, text}],                 // Sonnet questions, cached
          answers: {q1: "user answer", ...},       // user input, persisted
          eval: {score, feedback, passed},         // Sonnet eval, persisted
          status: 'notstarted'|'reading'|'passed'
        }
      },
      lab: {
        selectedOption: 'A'|'B',
        chat: [{role, text}],                       // ask-bar history
        status: 'notstarted'|'inprogress'|'done'
      },
      status: 'notstarted'|'inprogress'|'done'      // done = all concepts passed + lab done
    }
  }
}
```

## To extend

**Add a subtopic:** add an entry to `CURRICULUM` — the Opus generator handles the rest on first click.

**Change voice/quality:** edit the system prompt inside `generateSubtopic()`. Channel-voice list and quality bar are there.

**Swap models:** edit `OPUS` / `SONNET` constants at the top of the script.

**Update source list:** edit `SOURCES` const. Must match `reference_learning_catalog.md` in memory.

## Cost estimate

- Opus per subtopic: ~3K input + 10K output → ~$0.80
- Full curriculum (51 subtopics): ~$41 one-time
- Sonnet Q&A per concept: < $0.01
- Lab ask-bar turn with cache hit: ~$0.005

Paid compute for labs is avoided — all 2-option labs must run on free compute (Colab T4 / Kaggle P100 / Groq free / local CPU / Claude key the user already has).
